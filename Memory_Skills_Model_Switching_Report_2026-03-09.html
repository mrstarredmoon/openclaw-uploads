📋 Summary

This report documents the improvements made to how Candy handles memory and skills when switching between AI models, particularly between text-only and vision-capable models.

🔧 The Problem

  • When Warren switched from the default model to a text-only model (e.g., minimax-m2.5:cloud), Candy lost the ability to see images
  • Candy initially guessed incorrectly about images instead of using the proper tool
  • Memory files were fragmented (multiple files per day with timestamps)
  • No clear process for remembering context across model switches

✅ Solutions Implemented

1. Auto Image Analysis for Text-Only Models

Rule: When user sends ANY image on a text-only model, automatically run scripts/auto-image-analyze.py and include the description. Skip for vision-capable models.
  • Script uses OCR first, falls back to Vision (Nemotron Nano VL) if needed
  • Works seamlessly without user asking
  • Tested with toucan image — worked perfectly

2. Model Switch Awareness

Rule: When model changes, briefly re-familiarize with MEMORY.md/SKILLS.md to ensure smooth operation. Also reads today's + yesterday's memory files for context.
  • Candy checks her notes whenever model switches
  • Knows whether current model has vision or not
  • Applies appropriate tools automatically

3. Daily Memory Consolidation

Rule: One file per day (memory/YYYY-MM-DD.md), read today + yesterday on startup.
  • Consolidated 8 fragmented files into 1 clean file for today
  • Deleted old timestamped files
  • Now only 2 memory files: 2026-03-08.md and 2026-03-09.md
  • Ensures continuity across days and model switches
